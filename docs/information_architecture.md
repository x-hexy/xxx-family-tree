# 信息架构（information_architecture）

## 1. 主信息对象
- 个人（Member）
- 家庭单元（FamilyUnit）
- 单元关系（UnitRelation）

## 2. 页面结构
- 编辑页 `/`
- 左侧工具
- 中央视图（单元图）
- 右侧详情（当前仍以个人详情为主）
- 只读页 `/view/:token`

## 3. 主要交互路径
1. 搜索姓名 -> 命中个人 -> 定位到所在单元。
2. 点击头像 -> 选中个人 -> 右侧详情更新。
3. 拖拽单元卡 -> 同代锁 Y + 防重叠。
4. 从锚点拖线 -> 自动判定关系类型 -> 保存关系。
5. 点选关系线 -> 高亮 -> 删除或重连端点。

## 4. 可见性控制
- 视图模式：焦点树/父系/母系/全景。
- 关系线筛选：仅"父母/子女"开关。
- 代际筛选：可勾选显示特定代际。

## 5. 布局策略
- 自动布局：基于子树宽度逐代分配，含碰撞检测。
- 全景排序：父系左排、母系右排。
- 一键自动整理：清除手动位置回到最优布局。
- 拖拽持久化：仅全景模式保存手动位置。

## 6. 当前策略
- 迁移后默认无关系线，强调用户自主建线。
- 配偶关系已内化在家庭单元内，不再作为独立关系线。
- 夫妻卡自动按男左女右排序（基于姓名性别推断）。
- 连线交互采用"低压力模式"：上下锚点、高亮反馈、失败提示。
- 父系/母系视图支持跨分支连线。
- 右侧编辑面板可收起展开。
