# 视图规范（view_specification）

## 0. 页面入口与访问控制

| 路由 | 访问条件 | 说明 |
|------|---------|------|
| `/login` | 任何人 | 登录/注册页；已登录时自动跳转 `/` |
| `/` | 需登录（session 有效） | 编辑页；未登录时跳转 `/login` |
| `/view/:token` | 任何人（无需登录） | 只读分享页；token 无效或停用时显示提示 |

## 1. 模式定义
- `focus`：焦点树（2 跳邻域）。
- `paternal`：父系分支视图。
- `maternal`：母系分支视图。
- `overview`：全景视图。

## 2. 当前关系模型
- 关系类型：`parent_child`、`sibling`。
- 顶部可见性筛选：仅"父母/子女"开关（兄弟姐妹开关已移除）。
- `配偶`筛选已移除（夫妻已合并到同一家庭单元）。

## 3. 布局规则
- 全景：允许手工拖拽，位置持久化；父系排左、母系排右。
- 非全景：自动布局（基于子树宽度的 `calculateTreeLayout`）。
- 同代单元卡：Y 锁定。
- 同代单元卡：碰撞检测自动避让（重叠时向右推移 + 居中修正）。
- 一键"自动整理"：清除手动位置回到自动布局。

## 4. 连线规则
- 默认无自动关系线（迁移脚本不再生成）。
- 用户可自由创建/删除/重连。
- 仅上下两端锚点（简化操作）。
- 关系类型自动推断：
- 同代连接为 `sibling`。
- 不同代连接为 `parent_child`。
- 连接失败显示即时提示（非静默失败）。
- 边线交互宽度、连接半径、重连半径已增强。

## 5. 焦点联动
- 点击单元卡：选中焦点单元。
- 点击头像：选中个人并同步焦点单元。

## 6. 父系/母系规则

### 6.1 根节点定位（两段式搜索）
1. **祖先链搜索**：从焦点 unit 沿 `childToParents` 向上追溯所有祖先。
2. **全局回退**：若祖先中未命中根关键词，则搜索全部 unit。
- 父系根关键词：`["爷", "奶"]`
- 母系根关键词：`["外公", "外婆", "外"]`
3. 取辈分最高（generation 最小）的匹配 unit 作为分支根节点。

### 6.2 可见集合构建
- 从根节点 BFS 沿 `parent_child` 向下展开。
- 包含焦点 unit 的直接父 unit。
- **跨分支连线**：若已在可见集合中的 unit 有来自另一分支的 parent，也纳入可见集合。

### 6.3 主父节点机制
- 多父节点场景下（如爸爸妈妈同时被爷爷奶奶和外公外婆连线），每个子节点只归属一个主父节点。
- 优先选择父系（不含"外"字）parent，同辈按名字排序。

## 7. 编辑页 vs 只读页差异

| 能力 | 编辑页（`/`） | 只读页（`/view/:token`） |
|------|-------------|------------------------|
| 查看图谱 | ✅ | ✅ |
| 视图模式切换 | ✅ | ✅ |
| 搜索 | ✅ | ✅ |
| 导出 PNG/PDF | ✅ | ✅ |
| 编辑单元/关系线 | ✅ | ❌ |
| 上传头像 | ✅ | ❌ |
| 管理分享链接 | ✅ | ❌ |
| 登出按钮 | ✅ | ❌ |
