# 家族图谱应用设计文档（design）

## 1. 视觉风格
- 方向：卷轴质感、古典高雅、低噪点信息表达。
- 主色：宣纸底 + 墨色文本 + 朱砂强调。
- 动效：克制，强调状态反馈清晰性。

## 2. 交互设计（当前实现）

### 2.1 单元卡交互
- 点击卡片空白：选中单元。
- 点击头像/姓名：选中个人（右侧详情联动）。
- 夫妻卡自动按男左女右排序（基于姓名性别推断，无需手动互换）。

### 2.2 关系线交互
- 关系线可点击高亮、可删除。
- 支持拖动端点进行重连。
- 关系线命中宽度增大，降低选线难度。

### 2.3 连线创建机制
- 仅上下两端锚点，简化操作认知。
- 连线后根据代际差异自动推断关系类型：
- 同代连接：`sibling`
- 不同代连接：`parent_child`
- 连接失败时展示明确提示，避免用户无反馈。
- 锚点有连接态与可连接态高亮反馈。

### 2.4 布局交互
- 全景模式：允许拖拽并持久化位置。
- 同代约束：Y 轴锁定一致。
- 防重叠：同代单元卡自动避让（碰撞检测 + 向右推移 + 居中修正）。
- 一键"自动整理"：清除手动布局，回到最优自动排列。

### 2.5 面板控件
- 右侧编辑面板支持收起/展开（`▶/◀` 切换按钮）。
- 顶栏仅保留"父母/子女"连线可见性开关。

## 3. 视图规则
- 焦点树：2 跳邻域。
- 父系：从祖先链或全局搜索定位爷爷奶奶根节点，BFS 向下展开。
- 母系：从祖先链或全局搜索定位外公外婆根节点，BFS 向下展开。
- 父系/母系均支持跨分支连线（如爸爸妈妈 ← 外公外婆）。
- 全景：全量节点，父系排左、母系排右。

## 4. 智能布局算法
- `calculateTreeLayout`：基于子树宽度分配 X 坐标。
- 主父节点机制：每个子节点仅归属一个主父节点（父系优先），避免多父导致重复计算。
- 全景顶层排序：父系（不含"外"字）排左，母系（含"外"字）排右。
- 碰撞检测：同行节点按 X 排序，重叠时向右推移，推完后重新居中对齐父节点。

## 5. 头像体验策略
- 上传后即时预览。
- 上传失败降级保存，保证刷新后仍可见。
- 若 storage policy 正确，则对象存储正常落库。

## 6. 线条可读性策略
- 统一使用上下端连线，结合捆绑 Bézier 路径提升结构感。
- 选中关系线有明确发光高亮。
- 线条层级高于节点背景，减少"线被盖住"的感知问题。

## 7. 代际标签
- 每代左侧显示"谱 · 第 N 代"标签条带。
- 标签尺寸放大至 36px 印章 + 16px 文字，确保辨识度。
