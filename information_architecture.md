# 信息架构（information_architecture）

## 1. 主信息对象
- 个人（Member）
- 家庭单元（FamilyUnit）
- 单元关系（UnitRelation）

## 2. 页面结构
- 编辑页 `/`
- 左侧工具
- 中央图谱（单元图）
- 右侧详情（当前仍以个人详情为主）
- 只读页 `/view/:token`

## 3. 关键交互路径
1. 搜索姓名 -> 命中个人 -> 定位到所在单元
2. 点头像 -> 选中个人 -> 右侧详情
3. 拖动单元 -> 同代锁 Y + 防重叠
4. 拖线建关系 -> 选择关系类型 -> 保存

## 4. 可见性控制
- 视图模式：焦点树/父系/母系/全景
- 关系线过滤：父母子女、兄弟姐妹

## 5. 当前策略
- 迁移后默认无关系线，强调用户自主建线。
- 配偶关系已内化在单元卡内，不再作为独立关系线。
