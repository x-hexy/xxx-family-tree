# 家族图谱应用信息架构（IA）

## 1. IA 目标
- 在“单编辑者、亲友只读、免登录”的约束下，保证信息组织清晰。
- 让用户用最短路径完成三件事：编辑图谱、只读认亲、导出分享。
- 为后续多人协作预留导航与模块扩展位。

## 2. 角色与权限映射
- `编辑者`（你）：可访问编辑工作台的全部功能（成员、关系、布局、分享、导出）。
- `访客`（亲友）：只能通过分享链接访问只读视图（浏览、搜索、查看关系）。

## 3. 站点结构（Sitemap）
```text
家族图谱应用
├─ 编辑工作台 /
│  ├─ 图谱画布
│  ├─ 成员管理
│  ├─ 关系管理
│  ├─ 布局与筛选
│  ├─ 导出
│  └─ 分享设置
└─ 只读图谱 /view/:token
   ├─ 图谱浏览
   ├─ 搜索定位
   ├─ 成员详情
   └─ 关系理解辅助
```

## 4. 导航架构

### 4.1 编辑工作台导航
- 顶部全局栏（固定）：
- 家族标题、全局搜索、导出、分享、数据更新时间。
- 左侧工具导航（分区）：
- 成员操作区：新增/编辑/删除。
- 关系操作区：新增关系/删除关系。
- 视图控制区：按代际、分支筛选；重置视图。
- 中央主区域：
- 图谱画布（核心）。
- 右侧上下文面板：
- 选中成员详情、照片上传、关系快捷编辑、历史备注。

### 4.2 只读页导航
- 顶部简化栏：
- 家谱标题、搜索框、图例按钮、更新时间。
- 中央图谱区：
- 浏览与点击查看。
- 右侧详情区（或浮层）：
- 成员详情、与核心人物关系说明、称呼建议。

## 5. 信息对象与层级

### 5.1 一级信息对象
- `成员`：人脸 + 姓名 + 代际 + 基础说明。
- `关系`：父母/子女/配偶/兄弟姐妹。
- `图谱视图`：层级布局、筛选、缩放状态。
- `分享对象`：token、启停状态、有效期。
- `导出对象`：格式、范围、生成时间。

### 5.2 展示层级规则
- L1：节点卡片（姓名、人像、代际）。
- L2：详情卡（称呼、关系、简介）。
- L3：扩展信息（籍贯、备注、故事）。
- 原则：先让“他是谁”清楚，再展示“与我什么关系”，最后才是扩展背景。

## 6. 核心用户流

### 6.1 编辑建谱流（主流程）
1. 进入编辑工作台。
2. 新增核心成员（祖辈或自己）。
3. 持续新增家庭成员。
4. 建立关系并自动连线。
5. 为成员上传照片并确认人名对应。
6. 调整布局与筛选检查（树状代际层次）。
7. 导出或生成只读链接分享。

### 6.2 访客认亲流（主流程）
1. 打开 `/view/:token`。
2. 搜索姓名或点击人脸。
3. 查看详情卡的关系说明。
4. 沿关系连线继续查看上下代成员。

### 6.3 维护更新流（周期流程）
1. 编辑者打开工作台。
2. 新增/更新成员与关系。
3. 更新后重新导出或继续沿用原只读链接。

## 7. 页面级信息清单

### 7.1 编辑工作台
- 必备信息：
- 图谱主视图、工具区、详情区、导出/分享入口。
- 反馈信息：
- 操作提示、校验错误、保存状态。

### 7.2 只读图谱页
- 必备信息：
- 图谱、搜索、成员详情、关系说明。
- 限制信息：
- 明确“只读模式”，不出现编辑入口。

## 8. 命名与文案规范（关键）
- 关系名优先用用户熟悉词汇：父亲/母亲、配偶、子女、兄弟姐妹。
- 标注“与核心人物关系”时用完整句：`张三是你的二伯`。
- 关键按钮文案：
- `新增成员`、`建立关系`、`导出图谱`、`复制只读链接`。

## 9. IA 验收标准
1. 编辑者在不阅读说明的情况下，3 分钟内可完成首次建谱。
2. 访客在 30 秒内可找到任一成员并看懂与自己关系。
3. 只读页不出现任何误导性的可编辑入口。
4. 导出与分享入口始终可见、路径稳定。
5. 访客可通过“头像 + 姓名 + 代际”快速确认人物身份。
