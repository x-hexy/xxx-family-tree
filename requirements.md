# 家族图谱应用需求文档（requirements）

## 1. 项目目标
- 构建一个基于 Web 的家族图谱应用，帮助海外子女和亲友快速理解“谁是谁、彼此是什么关系、该如何称呼”。
- 应用由创建者（你）独立维护编辑，亲友通过只读链接浏览。
- 完成后的图谱可导出并分享，用于家庭沟通、节日聚会、代际传承。

## 2. 产品范围
- 平台形态：Web 应用（桌面优先，移动端可浏览）。
- 用户模型：
- `编辑者`：唯一管理员，负责所有新增、修改、删除、导出操作。
- `访客`：亲友，通过分享链接只读查看，无编辑权限。
- 账号策略：第一阶段不要求登录。

## 3. 关键场景
1. 编辑者录入家族成员信息与关系，形成完整图谱。
2. 访客通过链接打开图谱，按人脸和关系快速认亲。
3. 访客点击成员查看“与我关系、与核心家庭关系、称呼建议”。
4. 编辑者导出高质量图谱（图片/PDF）用于群分享或打印。

## 4. 功能需求（MVP）

### 4.1 家族成员管理
- 新增/编辑/删除成员。
- 每个成员支持字段：
- 基本信息：姓名、性别、出生年（可选）、世代、在世状态。
- 展示信息：头像/照片、一句话简介、常用称呼。
- 扩展信息（可选）：籍贯、备注。
- 支持为每个成员上传照片，并在图谱节点和详情卡中展示。

### 4.2 关系管理
- 支持关系类型：父母、子女、配偶、兄弟姐妹。
- 关系建立时做基础合法性校验（例如不可自指、不可形成明显冲突关系）。
- 图谱中自动连线并保持代际可读。

### 4.3 图谱浏览与检索
- 图谱支持平移、缩放、聚焦某成员。
- 支持按姓名搜索并高亮定位成员。
- 支持按分支/代际筛选显示。
- 图谱默认采用树状分层结构：祖辈在上、小辈在下、代际层次清晰。

### 4.4 成员详情卡
- 点击成员节点弹出详情卡，展示：
- 人像、姓名、称呼、简要信息。
- 与核心人物（如“我/我的孩子”）关系说明。
- 上下代直系关系概览。

### 4.5 分享与访问
- 生成只读分享链接。
- 访问链接无需登录即可查看。
- 支持“公开链接开关”（开启可访问，关闭即失效）。

### 4.6 导出
- 支持导出 PNG（高清）和 PDF（打印友好）。
- 支持导出范围：
- 当前视图导出。
- 全图导出（自动适配页面边界）。

## 5. 视觉与体验需求
- 设计风格：`家谱卷轴感`，古典、高雅、精致。
- 视觉元素：宣纸纹理背景、墨色文字、朱印点缀、细线分隔、卷轴式标题区。
- 字体风格：标题采用传统书卷气质字体，正文保持高可读性。
- 交互原则：
- 动效克制、过渡柔和，不喧宾夺主。
- 图谱信息密集但层级清晰，保证长辈与年轻用户都可理解。

## 6. 非功能需求
- 性能：中等规模家族（300-500 人）可流畅浏览与操作。
- 易用性：首次进入即可理解核心操作，关键动作（新增成员、连关系、导出）不超过 3 步。
- 可维护性：数据结构清晰，后续支持多人编辑与登录扩展。
- 兼容性：现代主流浏览器（Chrome、Edge、Safari 最新两个大版本）。

## 7. 验收标准（MVP）
1. 编辑者可完成成员与关系录入，生成可读图谱。
2. 访客通过链接可无登录查看图谱与成员详情。
3. 图谱在桌面端可稳定缩放、平移、检索定位。
4. 可成功导出清晰的 PNG 与 PDF 文件。
5. 页面风格满足“家谱卷轴感”且可用于正式家庭分享。
6. 图谱展示为树状代际布局，祖辈与小辈层次清晰可辨。
7. 每个人物节点可显示照片，编辑者可上传并更新照片。

## 8. 需求追踪编号（Requirements IDs）
- `REQ-F-001`：成员 CRUD（新增/编辑/删除）。
- `REQ-F-002`：关系管理（父母/子女/配偶/兄弟姐妹）与关系校验。
- `REQ-F-003`：图谱浏览（缩放/平移/聚焦）与搜索定位。
- `REQ-F-004`：只读分享链接（免登录、可启停）。
- `REQ-F-005`：导出 PNG/PDF（当前视图/全图）。
- `REQ-F-006`：图谱采用树状代际布局（祖辈在上，小辈在下）。
- `REQ-F-007`：成员照片上传，并在节点与详情卡展示。
- `REQ-UX-001`：整体视觉为“家谱卷轴感”，古典高雅精致。
