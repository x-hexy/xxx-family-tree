# 家族图谱应用需求文档（requirements）

## 1. 产品目标
- 构建一个基于 Web 的家族图谱应用，帮助家人快速理解“谁是谁、彼此什么关系”。
- 应用模式为“我编辑、亲友只读、免登录”。
- 图谱支持长期维护、分享链接访问、导出归档。

## 2. 当前范围（与实现一致）
- 数据模型采用 V2：以“家庭单元（夫妻卡/单人卡）”为关系节点。
- 图谱连线为“单元到单元”（非人到人）。
- 人员头像可上传并持久化；存储失败时有数据库降级兜底。
- 视图模式包含：焦点树、父系、母系、全景。

## 3. 角色与权限
- `编辑者`：唯一管理者，可编辑单元关系、拖拽布局、上传头像、分享、导出。
- `访客`：通过分享链接只读访问。
- 第一阶段不要求登录。

## 4. 功能需求

### 4.1 人员信息管理
- 支持人物信息编辑：姓名、代际、头像、简介等。
- 支持点击单元卡中的“个人头像/姓名”选中个人并查看详情。
- 支持头像上传并即时预览。
- 头像持久化策略：优先上传 Supabase Storage；失败时降级保存 data URL 到成员表。

### 4.2 家庭单元模型
- 支持家庭单元卡：
- 单人卡：仅 1 人。
- 夫妻卡：2 人同卡展示。
- 同一单元内左右位置固定，可一键“互换”夫妻左右槽位。

### 4.3 关系线编辑（单元级）
- 用户可自由创建单元关系线（默认不自动生成任何关系线）。
- 支持关系线选中、高亮、删除、端点重连。
- 支持关系类型：
- `parent_child`（父母/子女）
- `sibling`（兄弟姐妹）

### 4.4 图谱交互
- 支持平移、缩放、选中、搜索。
- 支持同代约束拖拽：同一代单元卡的 Y 坐标锁定一致。
- 支持同代防重叠：拖拽时与落点时自动避让，避免单元卡重叠。
- 交互优化：扩大连线命中区域与连接半径，提升可操作性。

### 4.5 视图模式
- `焦点树`：焦点单元为中心，显示 2 跳邻域。
- `父系`：基于焦点上游父母分支识别后展示父系分支。
- `母系`：基于焦点上游父母分支识别后展示母系分支。
- `全景`：展示全量单元，允许拖拽并保存位置。

### 4.6 分享与导出
- 支持只读分享链接（可启停）。
- 支持导出 PNG / PDF。

## 5. UI/UX 需求
- 风格：家谱卷轴感，古典高雅精致。
- 关系线交互必须可感知：可点击、可选中、可删除、可重连。
- 单元卡中“个人可选中”必须明确（点击头像有反馈）。

## 6. 非功能需求
- 性能：中等规模家谱可平滑操作。
- 可靠性：关系线创建/重连失败要可恢复，不产生脏状态。
- 一致性：需求、架构、设计、追溯文档必须与当前实现同步。

## 7. 需求 ID
- `REQ-F-001`：人物信息管理与详情编辑。
- `REQ-F-002`：家庭单元模型（单人卡/夫妻卡）。
- `REQ-F-003`：单元关系线创建、删除、重连。
- `REQ-F-004`：图谱交互（缩放/平移/搜索/选中）。
- `REQ-F-005`：同代锁 Y 与防重叠拖拽。
- `REQ-F-006`：视图模式（焦点树/父系/母系/全景）。
- `REQ-F-007`：头像上传与持久化（含降级兜底）。
- `REQ-F-008`：只读分享链接。
- `REQ-F-009`：PNG/PDF 导出。
- `REQ-UX-001`：卷轴风格与高保真交互体验。
