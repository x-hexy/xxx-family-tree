# 家族图谱应用需求文档（requirements）

## 1. 产品目标
- 构建一个基于 Web 的家族图谱应用，帮助家人快速理解“谁是谁、彼此什么关系”。
- 应用模式为“我编辑、亲友只读、免登录”。
- 支持长期维护、分享链接查看、导出图谱。

## 2. 当前范围（与实现一致）
- 数据模型采用 V2：以“家庭单元（夫妻卡/单人卡）”作为关系节点。
- 关系图主链路为“单元到单元”（非人到人）。
- 人物头像可上传并持久化；对象存储失败时支持 DB 降级兜底。
- 视图模式包含：焦点树、父系、母系、全景。

## 3. 角色与权限
- `编辑者`：唯一管理者，可编辑单元关系、调整布局、上传头像、分享、导出。
- `访客`：通过分享链接只读查看。
- 第一阶段不要求登录。

## 4. 功能需求

### 4.1 人物信息管理
- 支持人物信息维护：姓名、代际、头像、简介等。
- 支持点击单元卡中的“个人头像/姓名”选中个人并查看详情。
- 支持头像上传并即时预览。
- 头像持久化策略：优先上传 Supabase Storage；失败时降级保存 data URL 到成员表。

### 4.2 家庭单元模型
- 支持家庭单元：
- 单人卡：含 1 人。
- 夫妻卡：2 人同卡展示，自动按男左女右排序（基于姓名性别推断）。

### 4.3 关系线编辑（单元级）
- 用户可自由创建单元关系线（默认不自动生成关系线）。
- 支持关系线选中、高亮、删除、端点重连。
- 支持关系类型：
- `parent_child`（父母/子女）
- `sibling`（兄弟姐妹）

### 4.4 连线体验
- 每个单元卡提供上下两端锚点（source/target handle），简化连线操作。
- 连线后根据两端单元的代际自动推断关系类型：
- 同代连接判定为 `sibling`。
- 不同代连接判定为 `parent_child`。
- 连线失败必须提示可执行指引（例如"请拖到高亮锚点后再松开"）。
- 关系线需保持可见，不可被卡片视觉覆盖。

### 4.5 画布交互
- 支持平移、缩放、选中、搜索。
- 支持同代约束拖拽：同一代单元卡的 Y 坐标锁定一致。
- 支持同代防重叠：拖拽与落点时自动避让，避免单元卡重叠。

### 4.6 智能布局与自动整理
- 自动布局算法：基于子树宽度计算节点 X 坐标，逐代分配。
- 多父节点场景下，每个子节点只归属一个"主父节点"（优先父系）。
- 同行碰撞检测：同代节点若重叠，自动向右推移并重新居中。
- 全景模式下：父系（爷爷奶奶）排左侧，母系（外公外婆）排右侧。
- 支持一键"自动整理"：清除手动位置，回到自动布局并 fitView。

### 4.7 视图模式
- `焦点树`：焦点单元为中心，显示 2 跳邻域。
- `父系`：基于祖先链或全局搜索定位爷爷奶奶分支，BFS 向下展开。
- `母系`：基于祖先链或全局搜索定位外公外婆分支，BFS 向下展开。
- `全景`：展示全量单元，允许拖拽并保存位置。
- 父系/母系视图支持跨分支连线：若可见集合内的 unit 有来自另一分支的 parent，也纳入可见集合以渲染跨分支连线。

### 4.8 面板与控件
- 右侧编辑面板支持收起/展开。
- 顶栏线条可见性开关：仅保留"父母/子女"开关。

### 4.9 分享与导出
- 支持只读分享链接（可停用）。
- 支持导出 PNG / PDF。

## 5. UI/UX 需求
- 风格：家谱卷轴感，古典高雅精致。
- 代际标签（谱/第几代）清晰可辨，适当放大。
- 连线交互必须低学习成本。
- 锚点状态可视化明确：可连接、连接中、连接成功/失败可感知。
- 关系线选中状态必须明显，便于删除或重连。

## 6. 非功能需求
- 性能：中等规模家谱可平滑操作。
- 可靠性：关系线创建/重连失败可恢复，不产生脏状态。
- 一致性：需求、架构、设计、追溯文档必须与当前实现同步。

## 7. 需求 ID
- `REQ-F-001`：人物信息管理与详情查看。
- `REQ-F-002`：家庭单元模型（单人卡/夫妻卡，自动男左女右）。
- `REQ-F-003`：单元关系线创建、删除、重连。
- `REQ-F-004`：画布交互（缩放/平移/搜索/选中）。
- `REQ-F-005`：同代锁 Y 与防重叠拖拽。
- `REQ-F-006`：视图模式（焦点树/父系/母系/全景）。
- `REQ-F-007`：头像上传与持久化（含降级兜底）。
- `REQ-F-008`：只读分享链接。
- `REQ-F-009`：PNG/PDF 导出。
- `REQ-F-010`：智能布局与自动整理（子树宽度布局、碰撞检测、父系左排母系右排）。
- `REQ-F-011`：右侧面板可收起展开。
- `REQ-F-012`：父系/母系跨分支连线支持。
- `REQ-UX-001`：卷轴视觉与高保真交互体验。
- `REQ-UX-002`：低压力连线体验（上下锚点、自动类型推断、失败反馈）。
