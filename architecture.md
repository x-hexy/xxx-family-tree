# 家族图谱应用架构设计（architecture）

## 1. 架构目标
- 满足“单人编辑、亲友只读、免登录”的第一阶段目标。
- 保证图谱编辑流畅、浏览清晰、导出稳定。
- 为未来“多人协作 + 登录权限”预留扩展位。

## 2. 分层架构

### 2.1 前端层（Web SPA）
- 技术：React + TypeScript。
- 职责：
- 图谱渲染（SVG）。
- 成员与关系编辑交互。
- 分享链接管理与只读访问页面。
- 导出触发与结果下载。

### 2.2 API 层（应用服务）
- 技术：Node.js（Express/NestJS）。
- 职责：
- 提供成员、关系、分享、导出接口。
- 执行业务规则校验（关系冲突、数据合法性）。
- 统一错误处理与日志记录。

### 2.3 数据层
- PostgreSQL：
- 存储成员、关系、分享链接、布局坐标。
- 对象存储（S3 兼容）：
- 存储头像与导出文件。

## 3. 核心组件设计

### 3.1 Graph Engine（图谱引擎）
- 输入：成员列表 + 关系列表 + 可选手动坐标。
- 处理：
- 初始自动布局（树状代际分层，祖辈在上，小辈在下）。
- 手动拖拽后写回节点坐标。
- 输出：前端可渲染节点与边数据。

### 3.2 Avatar Pipeline（头像处理链路）
- 输入：本地图片文件或远端头像地址。
- 处理：
- 文件类型/大小校验。
- 生成前端即时预览（提升上传反馈）。
- 上传后保存头像 URL 并回写成员数据。
- 输出：节点卡片与详情卡统一头像数据源。

### 3.3 Relation Validator（关系校验器）
- 校验规则：
- 不允许 A -> A 自引用。
- 不允许同一对成员出现冲突关系。
- 删除成员时检查关系依赖并联动清理。

### 3.4 Share Gateway（分享网关）
- 生成高熵 Token 并绑定分享配置。
- 校验 Token 是否启用/过期。
- 只返回只读数据视图，不暴露写操作入口。

### 3.5 Export Service（导出服务）
- 导出模式：
- 客户端导出（轻量、实现快）。
- 服务端导出（稳定、可批量，作为增强）。
- 输出格式：PNG/PDF，支持全图与当前视图。

## 4. 页面与路由架构
- `/`：编辑工作台（仅编辑者使用）。
- `/view/:token`：访客只读图谱页。
- 前端状态划分：
- `entities`：成员、关系。
- `view`：缩放、平移、选中状态。
- `layout`：节点坐标、层级配置。
- `share`：分享配置状态。

## 5. 关键流程

### 5.1 编辑流程
1. 新增成员 -> 表单校验 -> 保存数据库。
2. 建立关系 -> 关系校验 -> 更新图谱。
3. 上传人物照片 -> 本地预览 -> 持久化头像地址。
4. 自动布局/手动调整 -> 保存坐标。

### 5.2 只读访问流程
1. 访客访问 `/view/:token`。
2. 后端校验 Token 状态。
3. 返回只读图谱数据。
4. 前端渲染并提供搜索/缩放/详情查看。

### 5.3 导出流程
1. 编辑者点击导出并选择格式。
2. 生成导出任务并渲染图谱快照。
3. 返回文件下载链接或直接下载。

## 6. 数据库结构建议（简版）
- `members(id, name, gender, birth_year, ... , created_at, updated_at)`
- `relationships(id, from_member_id, to_member_id, relation_type, created_at, updated_at)`
- `graph_layouts(id, member_id, x, y, version, updated_at)`
- `share_links(id, token, is_enabled, expires_at, created_at, updated_at)`
- 索引建议：
- `members(name)`
- `relationships(from_member_id, to_member_id)`
- `share_links(token unique)`

## 7. 扩展点预留
- 鉴权：后续可引入 JWT + 角色权限。
- 协作：成员变更审计表 + 乐观锁版本号。
- 历史：支持“时间快照”与版本回滚。
- 国际化：中英双语称谓系统，便于海外子女使用。

## 8. 实施阶段建议
1. Phase 1（MVP）：成员/关系/只读分享/基础导出。
2. Phase 2：布局优化、导出质量提升、性能调优。
3. Phase 3：多人编辑、登录鉴权、审计与版本管理。
