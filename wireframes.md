# 家族图谱应用线框文档（桌面端优先）

## 1. 线框说明
- 本文档为低保真结构线框，聚焦布局、信息层级、关键交互入口。
- 视觉风格由 `design.md` 落地（家谱卷轴感），此处不展开像素级样式。
- 默认分辨率基线：`1440 x 900`。

## 2. 页面 A：编辑工作台（`/`）

### 2.1 结构线框（ASCII）
```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 顶栏: [X氏家谱] [焦点树|父系|母系|全景] [☑父母/子女] [搜索框] [导出] [分享] │
├───────────────┬───────────────────────────────────────────┬──┬──────────────┤
│ 左侧工具栏    │              主图谱画布                     │▶│ 右侧详情面板  │
│               │                                           │◀│ (可收起展开)  │
│ [新增成员]    │  谱·第1代  [头像♂头像♀]   [头像♂头像♀]     │  │ [成员头像]    │
│ [建立关系]    │                │               │           │  │ 姓名/称呼     │
│ [自动整理]    │  谱·第2代  [头像♂头像♀]   [头像♂头像♀]     │  │ 基本信息表单  │
│               │                │               │           │  │ [照片上传]    │
│ [按代际筛选]  │  谱·第3代  [头像♂头像♀]   [头像♂头像♀]     │  │ [保存]        │
│ [按分支筛选]  │                                           │  │ [删除成员]    │
│ [重置视图]    │  (缩放控件 + 小地图 + 当前比例)            │  │              │
│               │                                           │  │              │
└───────────────┴───────────────────────────────────────────┴──┴──────────────┘
```

### 2.2 区域职责
- 顶栏：
- 全局级操作（视图切换、线条开关、搜索、导出、分享），始终可见。
- 左栏：
- 任务起点区，覆盖新增/关系/自动整理/筛选等高频动作。
- 中央画布：
- 图谱主工作区，支持缩放、平移、节点选择、拖拽。
- 画布右侧有 `▶/◀` 按钮控制右栏收起/展开。
- 右栏：
- 上下文编辑区，选中谁就编辑谁，避免弹窗打断。
- 支持收起以获得更大画布空间。

### 2.3 关键状态
- `空状态`：提示先创建“核心成员”，并显示一键引导按钮。
- `选中状态`：节点高亮 + 右侧展示详情。
- `关系编辑状态`：画布进入“选起点-选关系-选终点”流程提示。
- `头像上传状态`：右栏上传后，节点头像即时更新。

### 3.2 行为约束
- 不出现任何编辑按钮。
- 若 token 失效，显示失效页并引导联系编辑者。
- 搜索结果定位后自动聚焦并打开详情卡。

## 6. 组件到页面映射
- 编辑页：
- `FamilyNode`、`RelationEdge`、`GenerationFilter`、`AvatarUploader`、`PersonDrawer`、`ExportDialog`、`ShareDialog`
- 只读页：
- `FamilyNode`、`RelationEdge`、`PersonCard`

## 7. 交互优先级（实现顺序）
1. 图谱画布基础交互：缩放、平移、节点选中。
2. 成员新增与关系建立。
3. 只读页面搜索与详情查看。
4. 导出与分享弹窗。

## 8. 线框评审检查项
1. 是否一眼能找到“新增成员”和“导出图谱”。
2. 只读页是否零编辑入口。
3. 详情卡是否先显示“与核心关系”。
4. 大家族节点密集时，右栏信息是否仍可读。
5. 树状代际分层是否一眼可见（祖辈在上、小辈在下）。
6. 节点头像是否清晰且与姓名一一对应。
